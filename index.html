How Do Computers
Suggest Movies?

An Intuitive Look into the Movie Recommender

By Aashish Bedi

The "Librarian" Problem

The Overload

Imagine a library with 5,000 books but no labels. You want a book "like Inception," but the librarian doesn't speak your language.

The Solution

This project creates a Movie Translator. It converts descriptions, actors, and directors into a mathematical language the computer understands perfectly.

The Kitchen: Jupyter Notebook

Before a chef serves a meal, they experiment in the kitchen. Jupyter is our "Kitchen."

  • Mixing Ingredients: We combined the plot description, genres, and cast names.
  • Taste Testing: We checked if the data was messy (missing values) and cleaned it.
  • The Result: A perfect "Recipe" for what makes a movie unique.

Teaching the Computer "Grammar"

Computers are literal. To them, "Dance," "Dancing," and "Dancer" are three totally different things.

Porter Stemming (Simplification)

We use a tool called a Stemmer to chop off the endings of words. Now, the computer knows that all those words are just about "Danc."

Turning Movies into a Map

How does the computer "see" similarity? It places movies on a giant 5,000-dimensional map.

Bag of Words

We count the 5,000 most popular words. Every movie gets a "GPS Coordinate" based on these counts.

Distance = Similarity

If two movies have nearly the same coordinates, they are "neighbors" on the map. These are your recommendations!

Saving the Work: What is "Pickle"?

Calculating the map for 5,000 movies takes time and power. We don't want to do it every time a user clicks a button.

The "Freeze-Dry" Effect

Pickle allows us to "freeze" our calculated map into a small file (.pkl).

Instant Access

When the website starts, it just "thaws" the Pickle file. It’s like taking a pre-made meal out of the fridge instead of cooking from scratch.

The Messenger: FastAPI

The math lives in the basement (Server). The user is in the dining room (Website). How do they talk?

FastAPI is like a world-class waiter. You tell the waiter: "I want a movie like Batman." The waiter runs to the basement, asks the math engine, and brings the answer back to you in milliseconds.

The Interface: Streamlit

This is the actual "App" the user touches. We made it visual and easy to use.

  • Dropdown Menu: No typing required—just pick a movie.
  • Visual Posters: Using an API to fetch real posters from the internet so you don't just see a list of boring names.
  • Fast Results: Powered by the "Pickled" math we did earlier.

The Big Picture (Control Flow)

1. Build the Map (Notebook)
➔
2. Freeze the Map (Pickle)
➔
3. Serve the Map (FastAPI)
➔
4. Show the Map (Streamlit)

This "Pipeline" ensures that even though the math is complex, the user experience is as simple as Click and Discover.
